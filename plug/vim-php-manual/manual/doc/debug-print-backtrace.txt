debug_print_backtrace

(PHP 5, PHP 7)

debug_print_backtrace â€” Prints a backtrace

==============================================================================
*Description*

    void debug_print_backtrace ([ int $options = 0 [, int $limit = 0 ]] )~

debug_print_backtrace() prints a PHP backtrace. It prints the function calls,
included/required files and |eval|ed stuff.

==============================================================================
*Parameters*

{options}

    As of 5.3.6, this parameter is a bitmask for the following options:

                       debug_print_backtrace() options
                                Whether or not to omit the "args" index, and
    DEBUG_BACKTRACE_IGNORE_ARGS thus all the function/method arguments, to
                                save memory.

{limit}

    As of 5.4.0, this parameter can be used to limit the number of stack
    frames printed. By default ({limit}=0) it prints all stack frames.

==============================================================================
*Return* *Values*

No value is returned.

==============================================================================
*Changelog*

Version               Description
5.4.0   Added the optional parameter {limit}.
5.3.6   Added the optional parameter {options}.

==============================================================================
*Examples*

Example #1 debug_print_backtrace() example

<?php >
    // include.php file

    function a() {
        b();
    }

    function b() {
        c();
    }

    function c(){
        debug_print_backtrace();
    }

    a();

<?>
<?php >
    // test.php file
    // this is the file you should run

    include 'include.php';
<?>

The above example will output something similar to:

#0  c() called at [/tmp/include.php:10]
#1  b() called at [/tmp/include.php:6]
#2  a() called at [/tmp/include.php:17]
#3  include(/tmp/include.php) called at [/tmp/test.php:3]
